1) ASCII
2) ISO-8859-1(8비트)
3) EUC-KR(16비트,KSC-5601)
4) Unicode(16비트, ISO 10646)
 - 영어, 한글 모두 2바이트 표현
 - 일반적으로 통용되는 대부분의 국가의 문자를 담는다
 - 영어를 2바이트로 표현하기 때문에 비효율적이다.
 - 한글은 새로 정의했기 때문에 EUC-KR과 호환되지 않는다.
5) UTF-8
 - Unicode에 있는 문자를 규칙에 따라 1 ~ 4바이트로 변환한다.
 - ISO-8859-1(영어,숫자,특수문자): 1바이트로 표현한다.
 - 한글을 3바이트 포현

Exam0420

* 문자의 리터럴 - 문자 집합
1) 자바는 문자를 다룰 때 2바이트 값으로 다룸
2)  즉 각 문자는 규칙에 따라 0 ~ 65535까지 메모리에 저장
3) 각 문자를 표현하기 위해 정의딘 값을 '문자 코드'라 부름
4) 각 문자에 부여된 문자 코드의 집합을 '문자 집합'이라 부름
5) 자바가 사용하는 문자 집합은 '유니코드'임

* 유니코드
1) 전 세계의 모든 문자를 컴퓨터에 일관되게 처리할 목적으로 정의된 산업 표준 규칙
2) 문자를 2진수로 표현(encoding) 할 때, 두 가지 방식(UTF와 UCS)을 사용함
3) 자바는 메모리에 문자를 저장할 때 UCS(국제 문자 집합)을 사용하며, 외부 입출력 시 UTF를 사용
